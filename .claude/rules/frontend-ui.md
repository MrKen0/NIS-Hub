# Project Rule: Frontend UI

- Deliver mobile-first, accessible, responsive UI components.
- Use Tailwind CSS with design token classes (`bg-primary`, `text-secondary`, etc.).
- All screen states use skeleton or loading placeholders.
- No hard-coded pixel widths, use container and grid utilities.
- Ensure performance: images optimized with `next/image` and lazy loaded.
- Provide explicit aria labels and keyboard navigation.
